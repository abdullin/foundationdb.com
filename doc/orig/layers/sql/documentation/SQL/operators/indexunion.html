Union∞

Union operator combines the rows from two input streams. The output stream contains the contents of both input streams.

The Union Ordered operator merges the two streams, returning rows in the order based upon a merge field or fields. The input streams must be based on the same index. Duplicate rows are suppressed.

The Union All operator simily returns all rows from the first input stream, followed by the rows from the second. There are no guarantees on output order and duplicates are not eliminated.

Behavior∞

The output from the Union operator is formed by concatenating the first and second input streams.

Output∞

Rows of the first input stream followed by rows of the second input stream.

Assumptions∞

The row types on the two input streams are union-compatible, meaning they have the same number of fields and the corresponding field types match.

The Union Ordered assumes each input stream is ordered by its ordering columns.

Performance∞

This operator does no IO.

Memory Requirements∞

None
