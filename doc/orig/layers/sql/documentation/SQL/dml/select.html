SELECT∞

Purpose∞

The SELECT command selects a set of records from one or more tables or views.

Tables can be joined together with ANSI compatible inner and outer joins.

There are two versions of the SELECT statement:

  1. Standard ANSI SELECT
  2. Nested object SELECT

This page covers the syntax of both versions of the statement but only describes the functionality of the standard ANSI SELECT.

Syntax∞

SELECT [ DISTINCT ] column_expression [ , ... ]
  [ FROM table_expression [ join_expression [ ... ] ] ]
  [ WHERE where_expression
  [ GROUP BY column_expression [ , ... ] ]
  [ HAVING where_expression
  [ ORDER BY column_expression [ ASC | DESC ] [ , ... ] ]
  [ LIMIT { [ offset , ] row_count | row_count [ OFFSET offset ] } ]

where column_expression is:

{ column_name | expression } [ [ AS ] column_alias ]

where table_expression is:

{ table_name | view_name | ( subquery ) } [ [ AS ] table_alias ]

where join_expression is:

{
    *table_expression*
  | CROSS JOIN *table_expression*
  | [ INNER | LEFT | RIGHT ] JOIN *table_expression* ON *expression*
}

where where_expression is:

{ condition [ { AND | OR } condition ] [ ... ] }

Parameters∞

table_name
The name (optionally schema-qualified) of an existing table.
view_name
The name (optionally schema-qualified) of an eisting view.
subquery
Any valid SELECT query. For column expressions the limitation for the ANSI syntax is that a subquery must return exactly one field and one row. For nested objects multiple columns and multiple rows are allowed.
column_name
The name (optionally schema and table-qualified) of an existing column.
expression

Any valid expression. Valid expressions can only include column names that are available in the context of the query. Expressions can include column names, literals, functions and subqueries. Subqueries used in expressions in the SELECT part of the statement are not allowed to produce more than one row and one column except if the nested object syntax is used. Aggregate functions should only be used in combination with GROUP BY. If no GROUP BY is present the grouping is implicitly on all columns. Two special cases exist for expressions in the SELECT part of the statement:

  • returns all columns from all tables specified in the FROM (if table-qualified it returns all columns from the specified table).
  • returns all columns from a table including all columns from child tables. See SELECT Nested Objects for more details.
condition
An expression that returns a value of type boolean. Other types are allowed as long as they can be implicitly cast to the boolean type.
DISTINCT
Select on rows containing unique values.
FROM
FROM allows the specification of tables, views and subqueries to select columns from.
WHERE
WHERE is used to limit the number of rows that is returned in the result set.
GROUP BY
GROUP BY is used to reduce the result set by grouping together rows that have the same values in the columns specified in the GROUP BY clause. If GROUP BY is used, the columns in the SELECT part of the query have to be either included in the grouping columns or aggregate functions. If a column is specified that is not included in the grouping and no aggregate function is specified, the aggregate function FIRST is implied.
HAVING
HAVING is used to limit the number of rows produced by a grouping query by placing conditions on the grouped or aggregated values. HAVING is different from WHERE because it is executed after the grouping has taken place. Aggregate functions are allowed as part of the HAVING clause.
ORDER BY
ORDER BY is used to order the result set based on the specified column expressions. For each expression, ASC or DESC can optionally be specified to order the results in either ascending or descending order. If the order is not specified, the default order for each column is ascending. The collation for each column is used to determine the sort order.
LIMIT, OFFSET
LIMIT is used to limit the number of rows being output by the query. The row_count determines how many rows will be selected and the offset determines the starting point in the result set. A row_count of 10 and an offset of 5 will result in rows 5 through 14 being returned if they are available.

Notes∞

Using LIMIT can result in much better query performance in some cases because rows only need to be evaluated until offset + row_count is reached. The use of grouping and/or ordering can cause all records to be evaluated even if LIMIT is used. See the section on Optimizer and Query Execution for more information.

Using DISTINCT is the same as using GROUP BY on all selected columns.

Examples∞

Select all columns from all records in the orders table:

SELECT * FROM orders;

Select the number of customers with last name of Smith that placed an order on October 17th, 2012:

SELECT COUNT(distinct customers.customer_id)
FROM   customers
       INNER JOIN orders ON customers.customer_id = orders.order_id
WHERE  customers.last_name = 'Smith'
  AND  orders.order_date = '2012-10-17';
