fdbsqlcli Command Line Tool∞

The SQL Layer comes bundled with fdbsqlcli utility as a command line interface to communicate with any SQL Layer instance. fdbsqlcli is a terminal-based front-end that enables you to type in queries interactively, issue them to the SQL Layer, and see the query results. In addition, it provides a number of shorthand meta-commands.

How to start fdbsqlcli∞

After installing the SQL Layer, open a command line terminal window and start the fdbsqlcli CLI:

$ fdbsqlcli

or if your SQL Layer is installed on a different machine:

$ fdbsqlcli --host remote_host

You will be presented with a prompt similar to:

your_name=>

The command prompt now says your_name because this is the default schema it will use. To start with another schema:

$ fdbsqlcli schema_name

You will be presented with a prompt similar to:

schema_name=>

You can now type an valid SQL command or use the meta-commands enumerated below. Here are a few quick examples:

  • \lt
    • List all the tables in all schemas.
  • \ltS+
    • List all the tables in all schemas, including system tables, and also display extended information.
  • \lt getting_started.%
    • List all tables in the getting_started schema.
  • \l get%.%e%
    • List all (non-system) tables, views and sequences that are in any schema whose name starts with get and where the object has the letter e anywhere in it. The getting_started.test1 and getting_started.example tables both qualify.
  • \li test.%
    • List all (non-system) indexes for any table named test in any schema.
  • \li test%
    • List all (non-system) indexes from all tables and all schemas whose name starts with test.

Finally to exit type \q.

fdbsqlcli Options∞

The fdbsqlcli command line tool is used to interact with the SQL Layer:

user@host$ fdbsqlcli [options...] [schema]

Options∞

-c, --command <command(s)>∞

Execute command(s) and then exit.

-f, --file <filename>∞

Execute commands from the given file and then exit.

--on-error <CONTINUE|EXIT> <SUCCESS|FAILURE|SQLCODE|N>∞

Specify error handling with statements. See On-error handling for more details.

-o, --output <filename>∞

Specify the output file.

If the filename is not specified the results will be will be written to standard output.

-q, --quiet∞

Output only query results.

--skip-rc∞

Skip execution of rc file (~/.fdbsqlclirc) at startup.

-s, --schema <schema_name>∞

Specify the schema to connect to.

--help∞

Output help information and exit.

--version∞

Output version information and exit.

-h, --host <hostname>∞

Specify the host to connect to.

-p, --port <port_number>∞

Specify the port to connect to.

-u, --user <user_name>∞

Specify the user to connect as.

-w, --password <password>∞

Specify the password for the user.

-W, --password-prompt∞

Prompt for the password without echoing.

Tip

Also see Connection Details for more details.

schema∞

Specify the schema to connect to. This overrides any supplied --schema option.

fdbsqlcli Meta-Commands∞

Once in the command line, anything starting with an unquoted backslash is a meta-command that fdbsqlcli processes. Most of these commands are used to view metadata information and will generate SQL queries against the information_schema tables.

The format of a meta-command is the backslash, followed immediately by a command verb, and then by arguments. To include whitespace in an argument you can quote it with single quotes. To include a single quote in an argument, write two single quotes within single-quoted text.

As an example, to enumerate the user tables in the database, use the \lt (list-tables) meta-command:

schema-name=> \lt

And you will see a list of tables, similar to:

     Schema      |  Name |       Group Path
-----------------+-------+-----------------------
 getting_started | test1 | getting_started.test1
 getting_started | test2 | getting_started.test2
(2 rows)

The following meta-commands are defined:

In the commands below, supplying [S] will cause system objects to be included and [+] will display extended information about the objects. Both are optional.

For the describe and list commands, all arguments can be used to provide a pattern to filter the returned objects.

\c [SCHEMA [USER [HOST [PORT]]]]
Establishes a new connection to a SQL Layer using the given parameters. If a parameter is not given the value from the current connection is reused.
\conninfo
Get information about the current connection including user, host, port and schema in the format: user@host:port/schema
\d[S][+] [[SCHEMA] NAME]
Describe sequences, tables and views.
\dq[S][+] [[SCHEMA] NAME]
Describe sequences.
\dt[S][+] [[SCHEMA] NAME]
Describe tables.
\dv[S][+] [[SCHEMA] NAME]
Describe views.
\i FILE
Execute commands from a given file.
\o [FILE|-]
Send all query results to file. If FILE is omitted or -, send query results to standard output.
\l[S] [[SCHEMA] NAME]
List sequences, tables and views.
\li[S][+] [[[SCHEMA] TABLE] NAME]
List indexes.
\lq[S][+] [[SCHEMA] NAME]
List sequences.
\ls[S][+] [NAME]
List schemas.
\lt[S][+] [[SCHEMA] NAME]
List tables.
\lv[S][+] [[SCHEMA] NAME]
List views.
\timing
Toggle printing of query execution time.
\onerror [CONTINUE | EXIT [SUCCESS|FAILURE|SQLCODE|<N>]]
Sets the processing for handling errors with the statements. With no parameters it prints the current settings. See On-error handling for more details.
\?
Get help on the fdbsqlcli meta-commands.
\q [CODE]
Exit fdbsqlcli. If you supply the optional CODE value, an integer between 0 and 255, fdbsqlcli uses that as a exit code to the process.
\x
Toggle between regular and expanded output.
\null [STRING]
Show or set the representation of NULL values.
\t [ON|OFF]
Toggle or set tuple only output.
\f [STRING]
Show or set the format separator.
\a [ON|OFF]
Toggle or set alignment.

Meta-Commands filter pattern∞

Many of the commands accept a pattern in the form of [SCHEMA [NAME]] to filter the objects that are returned. A period (.) or space can be used to separate the schema and name. Enclose patterns in double-quotes to match names containing a period or space. Whenever the pattern is omitted completely, the commands display all objects that are visible.

In its simplest form, a pattern is just the exact name of an object. The characters within a pattern are case sensitive and can use SQL-style wildcards (% and _) to match multiple names. Precede wildcard characters with a backslash (\) to match names containing them literally.

On-error handling∞

The --on-error command line option and \onerror meta-command define the processing for errors. The error handling is for use in sql scripts, processed through the -f command line option or the \i meta-command. When on-error is enabled and fdbsqlcli encounters an error in execution of a SQL statement, processing stops and the fdbsqlcli sets an exit code.

The settings for the --on-error command line option and \onerror meta-command have two parts, a type and a status.

Error handling type is:

  • CONTINUE
    • The default setting. Processing does not stop when an error occurs, but continues to the next statement. Error handling status is always SUCCESS.
  • EXIT
    • Processing stops when an error is encountered. fdbsqlcli sets an exit code based upon the error handling status.

Error handling status is one of:

  • SUCCESS
    • The default settings. The exit code set is 0, implying a successful execution.
  • FAILURE
    • The exit code set is 1, implying a failed execution.
  • SQLCODE
    • fdbsqlcli determines the exit code from the specific error that occurs. This is a value between 1 and 255.
  • <N>
    • <N> is an integer between 0 and 255. When fdbsqlcli encounters an error this value is used as the exit code
