SQL Layer Client Tools Reference∞

Tools∞

fdbsqlcli – The command line interface to the SQL Layer.

fdbsqlload – Bulk loading from multiple formats into the SQL Layer.

fdbsqlload – Logical schema and data backup for the SQL Layer.

fdbsqlprotod – Protobuf message files for tables that utilize the Protobuf storage format.

Connection Details∞

All client tools need a host and port, along with user name and password if authentication is enabled, to connect to a SQL Layer instance. Values for those are specified in the following ways:

  • All command line tools take the following options:

    • -h, --host
    • -p, --port
    • -u, --user
    • -w, --password

  • If no command line option is specified, the following environment variables are used:

    • FDBSQL_HOST∞
    • FDBSQL_PORT∞
    • FDBSQL_USER∞
    • FDBSQL_PASSWORD∞

  • Otherwise, the following defaults are used:

    • Host: localhost
    • Port: 15432
    • User: <system user name>
    • Password: <blank>

  • To prompt for a password without echoing it:

    • -W, --password-prompt

  • Advanced options related to specific security setups:

    • --jaas-application-name
    • --jaas-config-file
    • --kerberos-service
    • --kerberos-kdc
    • --kerberos-realm
