Key-Value Store

The core of FoundationDB

The Key-Value Store is where all data in FoundationDB is stored. It can be distributed across many machines and has a simple but powerful key-value API with full ACID transactions.

View the Spec Sheet (PDF)

  • Multi-model

    The Key-Value Store's full multi-node ACID transactions allow developers to "layer" multiple data models on top of a single scalable database.

    See how this works →

  • Scalable & fast

    Key-Value Store provides amazing performance on commodity hardware and makes it incredibly easy to add more machines as your needs grow.

    See how it performs →

  • Fault-tolerant

    No more worrying about down time. Key-Value Store divides your data into chunks and distributes multiple copies of each chunk onto different storage servers.

    See why this matters →

Strong language support

We have supported language bindings for Python, Ruby, node.js, PHP, Java, Go, .NET, and C, and our community continues to build more.

    require 'fdb'
FDB.api_version 200
db = FDB.open
db.transact do |tr|
    # Read two values from the database
    a = tr['a']
    b = tr['b']
    # Write a key-value pair to the database
    tr['c'] = a + b
end

    # watch for changes to certain keys
def watching_loop(db, keys):

    @fdb.transactional
    def watch_keys(tr):
        return [tr.watch(k) for k in keys]

    while True:
        fdb.Future.wait_for_any(*watch_keys(db))
        yield
