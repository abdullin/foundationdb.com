fdbsqlprotod Command Line Tool∞

The client tool fdbsqlprotod obtains Protobuf message definitions from table-groups using the Protobuf storage format. The message definitions can be used to translate stored Protobuf messages into classes. A Python example of this process is given here.

Prerequisites∞

fdbsqlprotod comes with the SQL Layer Client Tools. See Installing Client Tools for installation details.

Usage∞

fdbsqlprotod will obtain the standard .proto files for a table-group:

user@host$ fdbsqlprotod [options...] groups...

Options∞

-d, --output-directory <directory>∞

Specify the output directory.

If the directory is not specified, the current working directory will be used.

-o, --output-file <filename>∞

Specify the output file name.

If the filename is not specified, results are written to standard output.

-s, --schema <schema_name>∞

Specify the schema to connect to.

--help∞

Output help information and exit.

--version∞

Output version information and exit.

-h, --host <hostname>∞

Specify the host to connect to.

-p, --port <port_number>∞

Specify the port to connect to.

-u, --user <user_name>∞

Specify the user to connect as.

-w, --password <password>∞

Specify the password for the user.

-W, --password-prompt∞

Prompt for the password without echoing.

Tip

Also see Connection Details for more details.

groups∞

Specify all groups that will be included in this dump file.

Table-Groups are specified using the root (i.e. parent-most) table name and must have been created using the Protobuf storage format.
