Data Typesâˆž

The FoundationDB SQL Layer supports the following data types.

Name Category Details
BIGINT Exact Numeric Signed integer with a range of -9223372036854775808 to +9223372036854775807.
BLOB Binary String Varying length, binary large object.
BOOLEAN Logic true or false.
CHAR[(length)] [CHARACTER SET cs] [COLLATE co] Character String Fixed-length character data. Default length, character set and collation are 1, utf8 and ucs_binary. See Localization for more details.
CHAR[(length)] FOR BIT DATA Binary String Fixed length binary data. Default length is 1.
CLOB Character String Varying length, character large object.
DATE Date/Time Year, month and day (YYYY-MM-DD).
DATETIME Date/Time Year, month, day, hour, minute and second (YYYY-MM-DD HH:MM:SS).
DECIMAL[(precision [, scale])] Exact Numeric Fixed-point number with a specified count of digits (precision) and fractional places (scale). Default precision and scale is 5 and 0. Maximum precision is 31. Scale must be <= precision.
DOUBLE Approximate Numeric Floating point number with a range of approximately 1e-307 to 1e+308 and at least 15 digits of precision.
GUID Binary String Globally Unique Identifier with an internal size of 128-bits.
INT Exact Numeric Signed integer with a range of -2147483648 to +2147483647.
REAL Approximate Numeric Floating point number with a range of approximately 1e-37 to 1e+37 and at least 6 digits of precision.
TIME Date/Time Signed hour, minute and second (HH:MM:SS) with a range of -838:59:59 to 838:59:59.
VARCHAR(length) [CHARACTER SET cs] [COLLATE co] Character String Varying-length character data. Default character set and collation are utf8 and ucs_binary. See Localization for more details.
VARCHAR(length) FOR BIT DATA Binary String Varying-length binary data.

Additional Typesâˆž

The following list contains type names with additional semantics.

Name Details
FLOAT[(p)] A floating point number with at least p binary bits of precision. Values between 1 and 23, inclusive, yield a REAL and between 24 and 52, inclusive, yield a DOUBLE. Default p is 52.
SERIAL INTEGER with automatically generated values. Limit one per table. Exact alias is INTEGER NOT NULL GENERATED BY DEFAULT AS IDENTITY. See CREATE TABLE for more details.
BIGSERIAL As SERIAL but with a BIGINT as the underlying data type.
INTERVAL An integer representing a period of time with units as YEAR, MONTH, DAY, HOUR, MINUTE, SECOND, YEAR_MONTH, DAY_HOUR, DAY_MINUTE, DAY_SECOND, HOUR_MINUTE, HOUR_SECOND, or MINUTE_SECOND, declared as INTERVAL value unit (e.g., INTERVAL 500 SECOND, INTERVAL 5 05:10:20 DAY_SECOND). Note: Unsupported as a column data type but can be added or subtracted from other date/time types.
