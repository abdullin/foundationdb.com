Buffer∞

The query processor uses the buffer operator to resolve the Holloween Problem. When you have a query that alters part or all of the primary key or columns in the hkey, such as:

UPDATE items SET item_id = item_id + 1

The operator processing needs to ensure the Update operation does not write the updated row back into the storage in a way that the row comes back through the operation execution a second time.

Behavior∞

The buffer operator reads all of the rows from the input into an in-memory buffer or into one or more temporary files when the cursor is opened. As the rows are requested, they are returned from the buffer until it is empty.

Notes∞

This uses the same underlying merge sort processor as the Sort operator, with the same perforamance characteristics.

Performance∞

The cursor open process reads all of the input rows from the input cursor and puts them into a memory buffer, if small enough, or writes them to one or more temporary files on a local dist (if larger). The output reads one row at a time from the memory buffer or row at a time from the temporary files until completed.

Memory requirements∞

The sort operation allocates a singe 64MB memory buffer. If the rows exceed the buffer, the buffer is written to a temporary file and the buffer reused. When reading, the process allocates a small IO buffer per temporary file for IO processing.
