Lookup∞

The lookup operator locates and returns related group rows based upon a branch of a group.

The lookup takes as input either an existing group row or an index entry row.

There are two basic types of lookup, plus a combined lookup;

  • Ancestor Lookup : return rows from the parent table(s)
  • Branch Lookup : returns the rows from the child table sub-tree under the input row
  • Group Lookup : returns both the parent rows and the child sub-tree.

Expected usages are to locate:

  • the group row corresponding to an index row. For example, an index on customer.name yields index rows which Lookup can then use to locate customer rows.
  • ancestors higher in the group. For example, given either an item row or an item index row, Lookup can be used to find the corresponding order and customer.
  • descendants lower in the group. For example, given either an order group row, Lookup can be used to find the corresponding items.

Behavior∞

As each input rows is read, the row combined with the query context is transformed to yield an hkey that will locate the corresponding row of the output row type.

For each ancestor type, the hkey is shortened if necessary, and the group table is then searched for record with that exact hkey.

For the branch lookup, the entire subtree under the base hkey is retrieved.

All the retrieved records are written to the output stream in hkey order (ancestors before descendants), as is the input row if KEEP_INPUT behavior is specified.

Performance∞

For each input row, Lookup does one random access for each parent type and as many sequential accesses as are needed to retrieve the entire child branch.

Memory Requirements∞

Lookup stores in memory up to (number of ancestors + 1) rows.
