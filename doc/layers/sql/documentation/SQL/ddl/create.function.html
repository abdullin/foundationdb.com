CREATE FUNCTION∞

Purpose∞

Creates a function or a reference to a function in an external library.

Syntax∞

CREATE [ OR REPLACE ] FUNCTION function name ( [ parameter_name ] parameter_type [ , ... ] )
   RETURNS return_type
   LANGUAGE language PARAMETER STYLE parameter_style
   [ EXTERNAL NAME '<jar>:<class>.<method>' ]
   [ AS 'quoted_definition' ]
   [ [ NOT ] DETERMINISTIC ]
   [ { CALLED | RETURNS NULL } ON NULL INPUT ]
   [ { NO SQL | READS SQL DATA | MODIFIES SQL DATA } ]
   [ { EXTERNAL SECURITY INVOKER | EXTERNAL SECURITY DEFINER } ]

Parameters∞

OR REPLACE
If a function of this name currently exists, the new definition replaces the current indention. Otherwise, attempting to create a duplicate function name results in an error.
function_name
The name (optionally schema-qualified) for the function.
parameter_name
The optional name for the each parameter to the function.
parameter_type
The data type of the value expected for this parameter.
return_type
The data type of the value returned by the function.
language

The language used for the writing of the function. While several JVM compatible languages have been tested, there is support for two:

JAVA
Function is written in Java and available as an external, loadable jar.
JAVASCRIPT
Procedure is written in Javascript and will be executed with the JVM Javascript engine.
parameter_style

Parameter style, so how the SQL translates the parameters of the procedure into the parameters of the procedures. There are two values:

JAVA
Inputs are passed as standard Java types (including primitives). For functions in languages other than Java, the procedure must define a named procedure to call.
VARIABLES:
Script is run directly without any requirement for a function (though they are allowed, of course). Inputs are passed by named variable binding. Only valid for JAVASCRIPT, or other scripting language procedures.
EXTERNAL NAME
The text of the function is defined in an external jar file, loaded into the JVM via the sqlj.install_jar procedure. If you are using a JAVASCRIPT procedure with a JAVA parameter style you need to include the EXTERNAL NAME parameter to allow the execution engine to determine which function in the quoted definition to call.
quoted_definition
An inline definition of the function. For a short, one line function you may enclose the function in single quotes. For longer functions, and any function spanning multiple lines, mark the beginning andend of the function with $$. There is no way to escape this marker, meaning your function definition can not use two dollar signs in a row.
NOT DETERMINISTIC
Deterministic functions return the same value for the same parameter values. By marking a function as deterministic, the optimizer may reduce the number of times the function is called. The default is NOT DETERMINISTIC.
CALLED ON NULL INPUT, RETURNS NULL ON NULL INPUT
IF RETURNS NULL ON NULL INPUT is specified and any of the parameters to the function are NULL, the function automatically returns a NULL value. The default is CALLED ON NULL INPUT which executes the function call with the NULL parameter values.

Notes∞

It is possible to use other JVM compatible languages (like Jython, JRuby, Groovy, Clojure) as function languages. Please contact FoundationDB for instructions on configuring your system to allow this.

Access to the local database from inside a script / Java method is via an embedded JDBC driver, using the standard embedded URL: jdbc:default:connection.

Functions written using JAVA language must use the parameter style JAVA. JAVASCRIPT Functions may use either JAVA (and define a function name) or VARIABLES.

Examples∞

A simple Javascript function using the JAVA parameter style. As a multi-line function it uses the $$ to delimit the start and end of the function:

CREATE FUNCTION js_function (x DOUBLE, y DOUBLE)
RETURNS DOUBLE LANGUAGE javascript PARAMETER STYLE JAVA
EXTERNAL NAME 'f' AS $$
  function f(x, y) {
    return x * y;
  }
$$;

A simple Javascript procedure using the VARIABLES parameter style:

CREATE OR REPLACE FUNCTION js_function (x DOUBLE, y DOUBLE)
RETURNS DOUBLE LANGUAGE javascript PARAMETER STYLE variables
AS 'x + y;';
