TRUNCATE TABLE∞

Purpose∞

The TRUNCATE command deletes all rows of a table.

Syntax∞

TRUNCATE TABLE table_name

Parameters∞

table_name
The name (optionally schema-qualified) of an existing table.

Notes∞

For tables that are the only table in the Table-Group, a TRUNCATE will complete very quickly. Otherwise, the processing is similar to a DELETE without a WHERE and must perform a scan of the entire table.

The TRUNCATE statement is currently transactional. This means that any concurrent write activity is likely to cause the truncate to fail due to conflicts. Additionally, a truncate of a large table is likely to fail before that due to the limit on long-running transactions. The following are suggested options to work around this limitation:

  • Use TRUNCATE GROUP when data in other tables should be removed as well.
  • ALTER the table out of the group, TRUNCATE and then ALTER the table back in.
  • Manually loop through DELETE FROM x WHERE id BETWEEN x AND y statements.

Examples∞

Removing all data from a table:

TRUNCATE TABLE customers;
