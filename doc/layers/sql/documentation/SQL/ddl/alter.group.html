ALTER GROUP∞

Purpose∞

Use the ALTER GROUP statement to merge or split Table-Groups.

Syntax∞

ALTER GROUP [ IF EXISTS ]
{
    DROP TABLE child_table_name
  | ADD TABLE child_table_name ( column_name [ , ... ] ) TO parent_table_name [ ( parent_column_name [ , ... ] ) ]
}

Parameters∞

IF EXISTS
Do not return an error if the table does not exist.
DROP TABLE

Remove a table from a Table-Group. The given table will become the root of a new Table-Group. All child tables are also moved to the new Table-Group.

It is not necessary to specify the name of the table group as each table can only be part of one Table-Group.

ADD TABLE
Add a table to an existing Table-Group. The columns for the parent table are optional and if omitted, the parent table’s PRIMARY KEY columns are used.

Notes∞

Each table is always in exactly one Table-Group. If a table isn’t added to a Table-Group when it is defined, it is implicitly made the root table of a Table-Group containing only itself. It is not possible to add a table to a group as a root table. Instead, you can add the child tables to the Table-Group of the root table.

All group operations can also be performed using GROUPING FOREIGN KEY clauses in CREATE TABLE and ALTER TABLE statements.

Example∞

Dropping a table:

ALTER GROUP DROP TABLE order_items;

Adding a table:

ALTER GROUP ADD TABLE order_customers (customer_id) TO customers (customer_id);
