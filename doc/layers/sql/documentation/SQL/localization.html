Localization∞

The CREATE TABLE statement allows per-column specification of sort order and character classification behavior.

Collation Support∞

The SQL Layer implements collations using ICU. Collations can be specified in two different formats using valid ICU locales and keywords.

  • locale[_case][_accent]
  • locale[_keyword=value]

case determines case sensitivity (ci or cs for insensitive and sensitive, respectively) and accent determines accent sensitivity (cx or co for insensitive and sensitive, respectively). keyword and value are a keyword-value pair; multiple keyword-value pairs can be included, separated with an underscore. The following are examples of valid collations:

  • en_us
  • en_us_ci
  • en_us_cs
  • en_us_ci_co
  • en_us_ci_cx
  • sv_se
  • zh_collation=stroke
  • zh_Hant_TW_collation=pinyin

Note that locales must be unambiguous. For example, fr_ci is an ambiguous collation specifier because ci could indicate a region (Ivory Coast) or case insensitivity. Instead, you could use fr__ci (in which ci is for case insensitivity), fr_ci_cs (in which ci is a region and case sensitivity is desired), or fr_ci_ci (in which ci is a region and case insensitivity is desired).

The SQL Layer also provides a binary collation, ucs_binary; this is used if no collation is specified when creating a table. The default collation can be set using the default collation configuration option.

The following example creates a table whose only column uses the en_us_ci collation.

CREATE TABLE t1
(
  c1 varchar(16) COLLATE en_us_ci
)

Character Set Support∞

The SQL Layer supports the following character sets:

  • latin1
  • utf8
  • utf16
  • iso_8859_1

If no character set is specified when creating a table, SQL Layer uses utf8. The default character set can be set using the default charset configuration option.

The following example creates a table whose only column uses the latin1 character set.

CREATE TABLE t1
(
  c1 varchar(16) CHARACTER SET latin1
)

Using both collation and character set∞

Note that if you define both a character set and a collation for an attribute, they must appear in that order:

CREATE TABLE t1
(
  c1 varchar(16) CHARACTER SET latin1 COLLATE en_us_ci
)

And not,

CREATE TABLE t1
(
  c1 varchar(16) COLLATE en_us_ci CHARACTER SET latin1
)
